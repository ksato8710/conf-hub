# ConfHub - リーンキャンバス & アクションプラン

---

## 1. リーンキャンバス

### 1. 課題 (Problem)

**Top 3 の課題:**

1. **カンファレンス情報が分散している** — connpass、TECH PLAY、Peatix、Doorkeeper、各社公式サイトなど複数のプラットフォームに情報が点在しており、エンジニア・デザイナーが網羅的に把握できない
2. **既存サービスのフィルタリングが貧弱** — 職種別、技術領域別、開催形式別、規模別、費用別など多軸での絞り込みができず、自分に関連するカンファレンスを見つけるのに時間がかかる
3. **カレンダー連携の不在** — 発見したカンファレンスを自分のスケジュールに取り込む導線がなく、参加検討から申込までの行動が分断される

**既存の代替手段と課題:**

| 既存サービス | 主な課題 |
|-------------|---------|
| connpass | 勉強会中心で検索性が低い。カンファレンス特化のフィルタリングがない |
| TECH PLAY | フィルタリング機能が貧弱。絞り込みの自由度が低い |
| IT勉強会カレンダー | UIが古く使いにくい。情報の鮮度にばらつきがある |
| Doorkeeper | 一覧性が低く、リストビューでの比較検討ができない |

---

### 2. 顧客セグメント (Customer Segments)

**アーリーアダプター（最優先）:**

- **ソフトウェアエンジニア（中堅〜シニア）** — 年に3回以上カンファレンスに参加し、技術トレンドのキャッチアップを重視する層。25〜40歳、東京・大阪在住が中心
- **UI/UXデザイナー** — デザインカンファレンスへの参加意欲が高いが、情報収集手段が限られている層

**二次ターゲット:**

- **プロダクトマネージャー** — 技術とビジネスの両面でカンファレンスに関心を持つ層
- **マーケター（テック企業所属）** — テックイベントでの登壇・出展を検討する層
- **カンファレンス主催者** — 自社イベントの集客チャネルを求めている企業・コミュニティ

**ペルソナ:**

> **田中さん（32歳・バックエンドエンジニア・東京）** — 普段はconnpassとTwitterで情報収集しているが、見落としが多い。月1回はカンファレンスに参加したいが、毎回探すのが面倒。Googleカレンダーで予定管理しており、ワンクリックで追加できると嬉しい。

### 3. 独自の価値提案 (Unique Value Proposition)

**ハイレベルコンセプト:**
> 「技術カンファレンスのための食べログ」

**UVP:**
> **日本のすべての技術カンファレンスを、職種・技術・形式・規模で自在に絞り込み、ワンクリックでカレンダーに追加できる唯一のサービス**

**差別化の3本柱:**

1. **カンファレンス特化** — 勉強会やミートアップではなく、規模のあるカンファレンスに焦点
2. **柔軟なフィルタリング** — 8軸（職種/技術領域/デザイン領域/形式/規模/参加費/地域/時期）での多次元フィルタリング
3. **カレンダー連携** — Googleカレンダー/iCalへのワンクリック追加で「発見」から「予定登録」までをシームレスに

---

### 4. ソリューション (Solution)

| 課題 | ソリューション |
|------|---------------|
| 情報の分散 | connpass API + TECH PLAY/Peatix/Doorkeeperスクレイピング + Twitter/X監視 + RSS + 自己投稿フォームによる**統合データ収集パイプライン**。AI分類によるカテゴリ自動付与 |
| フィルタリングの貧弱さ | 8軸の**多次元フィルタリングUI**（職種、技術領域、デザイン領域、形式、規模、参加費、地域、時期）。URLパラメータによるフィルタ状態の共有 |
| カレンダー連携の不在 | **Googleカレンダー追加ボタン**（MVP）、**iCalエクスポート**（Phase 2）。イベント詳細がカレンダーの説明欄に自動転記 |

**技術アーキテクチャ:**

- フロントエンド: Next.js 16 (App Router) + TypeScript + Tailwind CSS 4
- データベース: Supabase (PostgreSQL)
- ホスティング: Vercel
- データ収集: Node.jsスクリプト + Supabase Edge Functions

### 9. 圧倒的な優位性 (Unfair Advantage)

1. **データ収集パイプラインの先行構築** — 多層データ収集を最初に構築し、情報の網羅性で先行優位を確保
2. **AI分類による構造化データ資産** — 非構造化イベント情報をAIで自動分類。時間とともに価値が増大
3. **ネットワーク効果** — 主催者の自己登録 → 情報の質向上 → 利用者増加 → さらなる登録、の好循環
4. **ユーザー行動データの蓄積** — 技術領域別需要データが蓄積され、レコメンデーション精度や主催者向けインサイト提供に活用
5. **SEOドメインパワー** — カンファレンス関連キーワードでの早期上位表示
