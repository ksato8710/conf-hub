# カレンダーUI/UX ベンチマーク調査

## 調査サマリー

技術カンファレンス一覧サイト向けカレンダーUI/UXの主要サービスを調査した結果:

1. **リスト+カレンダーのハイブリッドが最適**: ミニカレンダー（ナビゲーション）+ イベントリスト（詳細）の組み合わせ
2. **時間バケット思考**: ユーザーは「今週」「来月」のような時間の塊で考える
3. **カテゴリ色分け**: 技術領域ごとの色分けが即座の視覚認識と記憶定着を改善
4. **ホバーリフト効果**: translateY(-4px) + shadow 増加がカード操作のフィードバックとして有効
5. **Material 3 Expressive**: 丸みのあるデザイン、太めの線が2026年トレンド

## ConfHub改善方針

### 1. ホーム画面: 週ストリップカレンダー
- 当週+翌週の14日間を横並び表示
- カテゴリ色ドットでイベントを視覚化
- 日付クリック → イベント表示
- 「カレンダーで詳しく見る →」リンクで /calendar へ誘導

### 2. カテゴリ色分けシステム
```
フロントエンド: blue-500    バックエンド: emerald-500
インフラ:      amber-500    AI・ML:      violet-500
モバイル:      pink-500     セキュリティ: red-500
データ:        teal-500
```

### 3. カレンダーページUI改善
- 日付セルにカテゴリ色ドットを表示（汎用青ドットから改善）
- イベントカードに左ボーダー色（カテゴリ識別）
- ホバーリフトエフェクト（translateY + shadow）
- スムーズなトランジション（250ms ease-out-cubic）
- 選択日のイベントパネルをよりリッチに

### 4. デザイントークン
- ホバー: `hover:-translate-y-1 hover:shadow-lg` (150ms transition)
- 角丸: `rounded-xl` (12px)
- 日曜: `text-red-500` / 土曜: `text-blue-500`
- 今日: 円形バッジ `bg-blue-600 text-white rounded-full`
- アニメーション抑制: `prefers-reduced-motion` 対応
